## 快速开始

### 前置条件

- 已部署 ComfyUI（本地或云端）
- 已安装 AstrBot
- 已将工作流 JSON 文件放入指定目录

### 安装插件

将插件文件夹放入 AstrBot 的 `data/plugins/` 目录，重启 AstrBot 即可。

### 基本用法

```
/画 一个白发Miku
```
在群内发送后，机器人自动生成图片并发送。

## 详细使用说明

### 🎨 `/画` — 文生图

```
/画 一个白发Miku
/画 16:9 一个白发Miku
```

- 第一个参数可选：比例（如 `9:16`、`1:1`、`4:3`）
- 剩余部分为提示词
- 支持直接发送纯文本触发（取决于工作流配置）

### 🖼️ `/图生图` — 图生图

```
/图生图 把背景改成红色
/图生图 0.7 把背景改成红色
```

- **引用图片**：回复/引用一张带图片的消息 + 发送命令
- **@群友头像**：@群友 + 发送命令（无引用图片时生效）
- 可选第一个参数为降噪值（`0.1` ~ `0.8`，默认由工作流决定）
- 降噪值越低，图片变化越小

### 🎬 `/图生视频` — 图生视频

```
/图生视频
```

- 需引用图片或 @群友获取头像
- 当前工作流需要是视频工作流（名称含 `视频`、`wan`、`ltx`、`animate`、`video`）
- 视频生成时间较长，请耐心等待

### 🎲 `/随机图` — 随机抽卡

```
/随机图
/随机图 5
```

- 只能在名称包含「抽卡」的工作流上使用
- 可选参数：抽卡张数（最多 10 张）
- 多张抽卡会依次执行并分别发送

### ✏️ `/编辑` — 图片编辑

```
/编辑 把这个猫换成狗
```

- 支持引用图片、@群友、上传图片（最多3张参考图）
- 图片会按顺序传入工作流的 LoadImage 节点
- 如果没有参考图会提示

### 📂 `/工作流` — 工作流管理

```
/工作流          → 显示所有工作流列表
/工作流 3        → 切换到编号3的工作流
/工作流 anime    → 搜索名称包含"anime"的工作流
/切换 anime      → 同上，快速切换
```

- 工作流按分类分组显示（画 / 图生图 / 图生视频 / 编辑 / 随机图）
- 选择后 10 秒内回复数字可切换
- 当前工作流标记 ✅

### 📐 `/比例` — 图片比例

```
/比例            → 显示所有可用比例
/比例 3          → 切换到编号3的比例
/比例 16:9       → 按名称切换
```

- 影响生成图片的宽高比
- 选择后 10 秒内回复数字可切换

### 🎚️ `/分辨率` — 质量等级

```
/分辨率           → 显示所有质量等级
/分辨率 2         → 切换到编号2的质量等级
/分辨率 1080p     → 按名称切换
```

- 等级：`480p`(SD) → `720p`(标清) → `1080p`(高清) → `2K`(超清) → `4K`(原画)
- 等级越高，图片越清晰但生成越慢

### 📊 `/队列` — 查看队列

```
/队列
```

- 显示 ComfyUI 当前队列状态
- 运行中任务数 + 等待中任务数

### ⏹️ `/停止` — 停止生成

```
/停止
```

- 停止当前用户提交的正在生成的任务
- 不影响其他用户的任务

### 🚀 `/执行` — 执行工作流

```
/执行 提示词
```

- 执行**当前工作流**，不限制分类
- 适用于未设置分类的工作流或需要临时覆盖分类的场景
- 与 `/画` 行为类似，但不会自动切换工作流分类
- 如果当前工作流需要图片输入，需引用图片或 @用户

### ↩️ `/撤回` — 撤回消息

```
/撤回
```

- 撤回本插件在该会话中最近发送的一张图片/视频
- 仅能撤回通过本插件 OneBot API 直接发送的消息
- AstrBot 标准发送的消息（无 message_id）无法撤回

### ❓ `/帮助` — 帮助

```
/帮助
```

- 显示当前工作流名称和所有可用命令

#### 🎚️ 界面透明度控制（2026-06-27 新增）

状态栏右侧的 ☁️ 滑块控制整个 WebUI 界面透明度：

- **范围**：30% ~ 100%（默认 100%）
- **实现方式**：CSS `color-mix(in oklch, ...)` 动态混合所有表面颜色
- **受影响的元素**：卡片（`--card`）、边框（`--card-border`）、遮罩层（`--bg-overlay`）、输入框背景（`--input-bg`）、悬浮态（`--card-hover`）、图标背景（`--icon-bg`）
- **不受影响的元素**：悬浮画廊按钮（`.gallery-fab`）、背景图片亮度、文字颜色、渐变、阴影
- 透明度值保存在浏览器 localStorage，刷新后自动恢复
- 拖动滑块时自动重置背景亮度滤镜（`--bg-brightness: 100`），避免透过透明界面看到亮度调整后的背景

### 命令关联

在 `workflow_config.json` 中配置命令与工作流的对应关系：

```json
{
  "__commands__": {
    "画": ["Anima动漫画图.json", "FLUX漫转真人.json"],
    "随机图": ["Anima随机抽卡.json"],

```

- 每个命令关联多个工作流，用户可以通过 `/工作流` 切换
- 工作流 JSON 的文件名（不含路径）作为唯一标识

## 部署模式

### 本地模式

```
AstrBot ──HTTP──→ ComfyUI (127.0.0.1:8188)
  │
  └── WebUI (127.0.0.1:8898)
```

- 所有组件运行在同一台电脑
- 上传图片保存在 ComfyUI input 目录
- 适合个人使用

### 云端模式

```
AstrBot (云端) ──HTTP──→ ComfyUI (云端或另一台服务器)
  │
  └── WebUI (0.0.0.0:8898)
       ↓ 公网访问
  用户浏览器
```

- AstrBot 部署在云服务器
- ComfyUI 可同机部署或远程
- WebUI 开启局域网访问后可通过公网 IP 访问
- 上传图片保存在 ComfyUI input 目录
- 输出图片通过 AstrBot 发送到 QQ

### 上传模式切换

通过 WebUI 设置面板的「上传模式」开关：
- **本地模式**：图片上传到 ComfyUI input 目录
- **远程模式**：图片上传到插件 upload 目录（ComfyUI 后端可通过网络访问时使用）

## 常见问题

### Q: 生成失败/超时

- 检查 ComfyUI 是否正常运行
- 检查 ComfyUI 地址配置是否正确
- 检查 ComfyUI 是否加载了所需的模型
- 视频生成可能需要更长时间，可在 `_process_and_submit` 的 `timeout_sec` 参数调整

### Q: WebUI 无法访问

- 默认绑定 `127.0.0.1`，仅本机可访问
- 需要在 WebUI 设置中开启「局域网访问」
- 云服务器需要开放对应端口的安全组/防火墙

### Q: 画廊没有图片

- 检查输出目录配置是否正确
- 图片需要是插件生成并保存到输出目录的
- 如果修改了输出目录，需要重启插件

### Q: 撤回不可用

- 撤回依赖 OneBot API 返回的 `message_id`
- 如果 AstrBot 使用 `context.send_message()` 标准发送（非 OneBot 直连），则无法撤回
- 检查插件日志中是否有 `记录发送图片` 字样

### Q: 进度条不显示阶段进度

- 此 ComfyUI 版本/分支不发送执行进度的 WebSocket 消息
- 进度条改为计时器模式，显示已运行时间
- 这是正常的，不影响图片生成

### Q: 多个用户同时使用

- 插件支持多用户并发
- 每个用户的任务独立排队
- `/停止` 仅停止当前用户自己的任务

#### 改动 2：comfyui — LLM 请求图片拦截降级（方案 C）

**问题**：DeepSeek V4 Flash 纯文本模型不支持 `image_url` 类型消息，导致用户发图时 LLM 调用失败。

**解决方案**：在 `main.py` 新增 `@filter.on_llm_request()` 钩子 `_on_llm_request`（第 1926 行），拦截 LLM 请求中的图片数据：

1. 遍历 `req.contexts`，检测 `image_url` 类型的 content item
2. 支持两种图片来源：
   - **base64 data URL**（QQ 图片 → AstrBot 转码）：解码后写入 upload 目录
   - **HTTP URL**：调用 `_download_image` 下载
3. 替换原 `image_url` 为纯文本：`[用户发送了图片，已保存到: path](请使用此路径调用工具)`
4. 安全兜底处理 `req.image_urls` 中未合并入 contexts 的图片
5. 清空 `req.image_urls` 防止上层再次组装
